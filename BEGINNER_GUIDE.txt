Example server.js:
```javascript
import express from 'express';
import mongoose from 'mongoose';
import dotenv from 'dotenv';
import cors from 'cors';

dotenv.config();
const app = express();

app.use(cors());
app.use(express.json());

// Test route
app.get('/api/test', (req, res) => {
  res.json({ message: 'API is working!' });
});

// MongoDB connection
mongoose.connect(process.env.MONGODB_URI)
  .then(() => console.log('MongoDB connected'))
  .catch(err => console.error(err));

const PORT = process.env.PORT || 5000;
app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});
```

DEBUGGING TIPS FOR BEGINNERS
================================================================================

WHEN BACKEND DOESN'T WORK
--------------------------------------------------------------------------------
1. Check if server is running (look for "Server running on port...")
2. Check MongoDB connection (look for "MongoDB connected")
3. Use console.log() to see what data you're receiving
4. Check Postman response for error messages
5. Look at terminal for error messages

WHEN FRONTEND DOESN'T WORK
--------------------------------------------------------------------------------
1. Open browser console (F12) and check for errors
2. Check if API calls are successful (Network tab)
3. Use console.log() to see component state
4. Check if backend is running
5. Verify API endpoint URLs are correct

COMMON ERRORS & SOLUTIONS
--------------------------------------------------------------------------------
Error: "Cannot GET /api/..."
Solution: Check if route is defined in backend

Error: "CORS policy blocked"
Solution: Add cors() middleware in backend

Error: "Cannot read property of undefined"
Solution: Check if data exists before accessing it

Error: "MongooseError: Operation buffering timed out"
Solution: Check MongoDB connection string

Error: "JWT malformed"
Solution: Check if token is being sent correctly
